Phase 1
Discovery
Jun 14–19
Key Activities
  • Platform orientation — independent read-through and exploration
  • Facilitator conversations — learner needs, common misunderstandings, content gaps
  • Broader facilitator survey — 3–5 question form to validate patterns at scale
  • Content inventory — catalog what exists on the support site and platform
  • Review existing video assets for extractable language
  • Synthesize findings across conversations, survey responses, and inventory exploration into a rough map and a short list of high-confidence build candidates
⏱️
Hard Cap Mindset
The output of this phase isn't completeness — it's a rough map of what exists, the obvious gaps, and 1–2 high-confidence things worth building. If the week ends and you're still "figuring it out," it's gone too deep.
Phase 2
Design + Build Start
Jun 22–24
Key Activities
  • Content mapping — what exists, what needs to be built, where things link or extend
  • Categorize content as: Link (point to existing), Companion (extend existing), or New Build
  • Bring 2–3 tiered directions early in the phase
  • In parallel: begin building on anything clearly Safe or Stretch — drafts, outlines, page structure
  • Early draft of mapping delivered — Jun 26
  • EOP Target a largely complete map, and at least one real asset in draft form — not just planned.
↓ Tiered Ambition Framework — applied in this phase

Before bringing content ideas to review, each will be categorized by ambition level. This prevents good ideas from being self-filtered out before leadership has a chance to weigh in — and provides a menu, not just a single recommendation.

🟢 Safe — existing content, light lift, clear precedent
🔵 Stretch — new content, moderate build, fits expected scope
🟣 Bold — new format, feature, or approach; Pendo territory

Phase 5
QA & Review
Jul 2–3
Key Activities
  • Consistency, tone, and accuracy pass across all content
  • Facilitator QA — loop in trusted facilitators to flag anything that doesn't reflect how the platform actually works
  • Final sign-off before handoff to tech team
  • Revision cleanup — address feedback before pages are considered ready for implementation
✅
Final Reviews
By this point, review has already been happening throughout Phases 2–4 — this phase confirms everything is ready, not where direction gets decided. Focus is consistency and accuracy. If you're still debating what to build here, you're behind.

Tiered Ambition Framework
The Tiered Ambition Framework
Applied in Phase 2 · Design

During the Design phase, every content idea will be assigned a tier before going to review:

Safe
  • Existing content or a light-lift adaptation with clear precedent. Low risk, low resource, high confidence it will be accepted as-is.
🟢Looks like
  • Linking to an existing support page already covering the topic
  • Repurposing language from a vetted video into a written page
  • A companion page that extends something already built, staying close to the original
  • A feature explainer that mirrors the platform's own existing documentation
Stretch
  • New content that doesn't exist yet but fits comfortably within expected scope. Moderate build effort; the idea itself isn't a reach, just the execution.
🔵Looks like
  • Creating scenario-based content that walks through how two different assessment types might interact
  • Drafting a facilitator-facing reference page that doesn't exist anywhere currently
Bold
  • Genuinely new in format, feature, or approach. Outside established precedent. Requires leadership sign-off before investing in. Pendo territory ideas.
🟣Looks like
  • Embedding an interactive element within the platform page itself (e.g. a "?" help trigger)
  • Placing content inside the platform UI rather than on a support page
  • Introducing a format or content type that hasn't been done before in this context
